# GLOSSARY.md Format

`GLOSSARY.md` lives at `~/.openvista/learning/GLOSSARY.md`. It is the canonical language for this learning workspace. All explainers, exercises, and learning records should adhere to its terminology.

## Structure

```markdown
# [语言] 术语表

{这门语言的简要描述}

## Terms

**术语名**:
定义。一两句话，说明术语是什么，而非它做什么或怎么做。
_Avoid_: 同义词/避免使用的词

**另一个术语**:
定义。
```

## Rules

- **只在用户理解后才添加。** GLOSSARY 是压缩知识的记录，不是用户阅读学习的字典。如果用户刚接触某个概念，等他们能正确使用后再提升到这里。
- **有主见。** 当一个概念有多个名称时，选最好的，列出其他作为别名避免混淆。这是语言压缩的方式。
- **定义要紧凑。** 一两句话。定义术语是什么，不是它做什么。
- **用 GLOSSARY 自己的术语。** 一旦术语进入 GLOSSARY，在各处都使用它——包括在其他定义中。这使得复杂术语后来更容易理解。
- **按自然群分组。** 如 ## 发音、## 语法、## 词汇。扁平列表也可以当术语不多时。
- **明确标注歧义。** 如果术语在更广泛的领域中使用松散，注明决议：在这份文档中，"X" 始终指 Y。
- **随理解加深而修订。** 第一周写的定义到第六周可能就错了。在原地更新；不留陈旧条目。
