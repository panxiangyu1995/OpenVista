# MISSION.md Format

`MISSION.md` lives at `~/.openvista/learning/MISSION.md`. It captures the _reason_ the user is learning this language. Every teaching decision should trace back to this document.

## Template

```markdown
# Mission: [语言] 学习

## Why
{1-3 sentences. 用户学习这门语言的现实目标。学成后生活会有什么改变？避免抽象表述如"我想理解X"，要追向底层结果。}

## Success looks like
- {一个具体的、可观察的学习成果}
- {另一个具体成果}

## Constraints
- 时间：{每周可投入时间，每天哪个时间段}
- 目标：{学习多久，达到什么水平}
- 其他约束：{已有承诺、学习偏好等}

## Out of scope
- {用户明确不想追求的相邻主题——保护近侧发展区}
```

## Rules

- **One mission per workspace.** 如果用户想学两门不相关的语言，这是两个独立的工作区。
- **具体优于抽象。** "3个月内能进行基础对话" 比 "学点俄语" 好。
- **追问模糊表述。** 如果用户说不清楚为什么，访谈后再写。坏的 mission 比没有更糟。
- **现实变化时修订。** 用户的目标会变，更新此文件——不要让过时的 mission 指导后续 sessions。
- **保持简短。** 如果 MISSION.md 超过一屏，它就不再是指南针而是计划书了。
